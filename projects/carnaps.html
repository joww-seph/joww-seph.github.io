carNaps

A sensor-based search and rescue robotic system

An autonomous robotic system designed to detect obstacles and collect objects using multiple sensors and a motorized claw mechanism — built for search and rescue, autonomous delivery, and object collection applications.

Project PDF
carNaps robot — front view

Overview

carNaps is an autonomous robotic system designed to detect obstacles and collect objects using multiple sensors and a motorized claw mechanism. The robot utilizes three ultrasonic sensors to measure distance and identify obstacles in its surroundings, while infrared sensors assist in identifying diagonal proximity and object detection.

Built around an Arduino R3 UNO and driven by an L298 motor driver, carNaps combines sensor fusion, autonomous navigation, and motorized manipulation into a single cohesive platform suitable for real-world search and rescue scenarios.

Hardware Components

Qty Component
1× Arduino R3 UNO
1× L298 Motor Driver
3× Ultrasonic Sensors
3× IR Proximity Sensors
4× DC Motors
3× Servo Motors

Key Features

  • Multi-sensor obstacle detection — 3 ultrasonic sensors cover front and side distances; IR sensors handle diagonal proximity
  • Autonomous navigation — real-time pathfinding that reacts to the environment without human input
  • Robotic claw manipulation — motorized claw driven by servo motors for object grasping and collection
  • Efficient motor control — L298 driver manages 4 DC motors for independent wheel drive
  • Integrated sensor processing — all sensor inputs fused and processed on-board by the Arduino

Applications

  • Search and rescue operations
  • Autonomous delivery
  • Warehouse object collection
  • Robotics education and research

What I Learned

Building carNaps taught me how to coordinate multiple sensor streams simultaneously — ultrasonic distance readings, IR proximity signals, and servo feedback — all processed in real time on a single microcontroller. The biggest challenge was preventing sensor interference and writing an obstacle-avoidance algorithm that stayed stable across varying environments.

Adding the robotic claw pushed me into servo motor timing and mechanical control, areas I hadn't explored before. It also highlighted the importance of power management: running four DC motors, three servos, and six sensors off a shared supply required careful load balancing to avoid voltage drops that caused erratic behavior.
