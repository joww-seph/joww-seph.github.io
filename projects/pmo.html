PMO — Power Monitor Operator

A residential energy monitoring and protection device built around an ESP32 microcontroller. Real-time electrical monitoring with a BMO-inspired animated face, cloud analytics, and AI-powered insights. Work In Progress.

Add a screenshot here

Overview

PMO (Power Monitor Operator) is a residential energy monitoring and protection system built around an ESP32 microcontroller with full IoT cloud integration. It measures 13 real-time electrical metrics via a PZEM-004T v3.0 power sensor and provides automated circuit protection, remote relay control, AI-powered energy analytics, and intelligent appliance identification — all in a single device that operates both offline (local touchscreen) and online (web dashboard).

The device targets Philippine households: it tracks live cost and CO₂ output, trips a relay automatically on 7 fault conditions, identifies which appliances are active through NILM signature matching, and generates twice-daily energy reports using Google Gemini AI. A four-page web dashboard at pmo.infinityfree.me provides remote relay control, live charts, historical logs, and NILM management — refreshing every second without WebSockets.

Firmware Features

  • Animated Face Interface (TFT): BMO-inspired animated face with states: idle (persistent), touch reaction (2s), and blink (150ms every 7s)
  • 16×2 LCD Metric Display: Rotates through 8 screens every 3 seconds (voltage, current, power, frequency, cost, CO₂, safety margin, etc.)
  • WiFi Manager: Custom touchscreen provisioning UI with on-screen keyboard, signal strength indicators, and SPIFFS persistence
  • Real-time Sensor Loop: PZEM-004T polled every 2 seconds over UART; derives cost, CO₂ footprint, load type, power factor, and more
  • Relay Protection: 7-condition fault detection (undervoltage, overvoltage, overcurrent, overpower, under/over-frequency, low PF) with hysteresis and auto-restore
  • Cloud Logging: HTTPS POST via WiFiClientSecure with AES-128-CBC cookie bypass for InfinityFree hosting
  • OLED Display: Shows current time, date, relay status, and active trip reason (SSD1306)
  • Dual-Core FreeRTOS: Core 1 handles sensor reads and protection at 2s; Core 0 runs async cloud sync at 3s — semaphore-protected shared payload

Metrics Engine

The PZEM-004T v3.0 sensor is polled every 2 seconds, delivering raw readings (voltage, current, power, energy, frequency, power factor). These are processed to derive:

  • Apparent Power (S): V × I (VA)
  • Reactive Power (Q): √(S² − P²) (VAR)
  • Load Type: Classified by power factor (resistive, mixed, inductive, reactive)
  • Cost/Hour: (P / 1000) × ₱13.8161/kWh
  • Wasted Power: S − P (W)
  • Safety Margin: ((16 − I) / 16) × 100 (%)
  • CO₂ Equivalent: energy × 0.6032 kg/kWh
  • Frequency Deviation: |f − 60| Hz

Constants: 220V nominal @ 60Hz, 16A circuit rating, ₱13.8161/kWh electricity cost (Meralco), 0.6032 kg CO₂/kWh emission factor.

Relay Protection System

The relay is normally energized. It trips OFF when any fault occurs and auto-restores after a 5-second cooldown. All 7 conditions use hysteresis to prevent relay chatter:

Protection Trip Threshold Restore Threshold
Undervoltage < 195 V ≥ 205 V
Overvoltage > 250 V ≤ 240 V
Overcurrent > 15 A ≤ 13.5 A
Overpower > 3200 W ≤ 3000 W
Under-frequency < 57 Hz ≥ 58.5 Hz
Over-frequency > 63 Hz ≤ 61.5 Hz
Low Power Factor < 0.50 (P > 100W) ≥ 0.65 or P < 50W

Cloud Backend & Web Dashboard

Hosting: InfinityFree (HTTPS at pmo.infinityfree.me) | Database: phpMyAdmin | Stack: PHP, Vanilla JS, Chart.js

The ESP32 posts timestamped readings every 2 seconds. The four-page dashboard polls the API at 1 Hz with stale-connection detection (visual lock after 5 missed updates):

  • Dashboard — relay control panel, AI analytics card (letter grade A–F, ₱ bill prediction, 3 recommendations)
  • Monitor — 13 live metric cards with dynamic color thresholds (green → amber → red)
  • History — 4 real-time Chart.js line charts (Voltage, Power Triangle, Current/PF, Frequency) + 30-reading scrollable log
  • NILM — appliance registration modal (30s sampling), live detection display with confidence scores

AI Analytics: Google Gemini 1.5 Flash generates energy reports twice daily (6 AM & 6 PM, Asia/Manila), aggregating daily stats into a letter grade, projected month-end bill, and 3 household-specific recommendations. Results are cached with a 5.5-hour TTL.

NILM (Non-Intrusive Load Monitoring): Users register appliance power signatures through a 30-second sampling flow. Detection runs an exhaustive subset search against live metrics and returns matched device names with confidence scores — no additional sensors required.

Tech Stack & Key Dependencies

Firmware (C/C++ on Arduino Core v3.x):

  • TFT_eSPI (ILI9488 rendering + XPT2046 touch)
  • LiquidCrystal_I2C (16×2 LCD control)
  • PZEM004Tv30 (power sensor driver)
  • Adafruit_SSD1306 (OLED rendering)
  • ArduinoJson (JSON serialization)
  • WiFiClientSecure + HTTPClient (HTTPS communication)
  • mbedtls/aes.h (AES-128-CBC for InfinityFree cookie bypass)
  • SPIFFS (persistent storage)

Web Backend: PHP + phpMyAdmin + HTML/CSS/Vanilla JS + Chart.js

AI: Google Gemini (energy analytics) | Optional ML: Edge Impulse (person detection on ESP32-CAM)
