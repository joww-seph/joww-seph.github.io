PMO — Power Monitor Operator

A residential energy monitoring and protection device built around an ESP32 microcontroller. Real-time electrical monitoring with a BMO-inspired animated face, cloud analytics, and AI-powered insights. Work In Progress.

Add a screenshot here

Overview

PMO (Power Monitor Operator) is a residential energy monitoring and protection system built around an ESP32 microcontroller with full IoT cloud integration. It measures 13 real-time electrical metrics via a PZEM-004T v3.0 power sensor and provides automated circuit protection, remote relay control, AI-powered energy analytics, and intelligent appliance identification — all in a single device that operates both offline (local touchscreen) and online (web dashboard).

The device targets Philippine households: it tracks live cost and CO₂ output, trips a relay automatically on 7 fault conditions, identifies which appliances are active through NILM signature matching, and generates twice-daily energy reports using Google Gemini AI. A four-page web dashboard at pmo.infinityfree.me provides remote relay control, live charts, historical logs, and NILM management — refreshing every second without WebSockets.

Problem Statement

Philippine households face critical energy challenges:

  • Poor Energy Visibility: No real-time consumption tracking beyond monthly bills
  • Safety Risks: Overcurrent, voltage surges, and electrical faults go undetected
  • Cloud Dependency: WiFi-only monitors become useless during outages
  • Ineffective Cost Reduction: Generic energy tips don't match actual household usage patterns
  • Appliance Identification: Impossible to identify which devices consume the most power without sub-metering
  • Safety Monitoring: No visibility into power factor, frequency deviation, or CO₂ footprint without an electrician

PMO solves these problems with a single device that works offline and online, combining hardware safety with AI-powered insights.

System Architecture

Three-Tier Design: Local Hardware → Cloud API → Web Dashboard

  • Hardware Layer: ESP32 core reads PZEM sensor at 2s intervals, evaluates 7 protection criteria, updates 3 display types, manages relay switching
  • Cloud Layer: PHP backend on InfinityFree receives sensor data, caches relay commands, runs Gemini AI analytics at 6 AM/6 PM, manages NILM appliance registry
  • Frontend Layer: Vanilla JS dashboard polls cloud API at 1 Hz, renders real-time charts via Chart.js, provides relay control and NILM registration UI
  • Dual-Core Firmware: Core 1 (sensor + protection @ 2s), Core 0 (cloud sync @ 3s) with FreeRTOS semaphore synchronization

Firmware Features

  • Animated Face Interface (TFT): BMO-inspired character with idle, touch reaction (2s), and blink animations (150ms every 7s) — emotional feedback for user interactions
  • 16×2 LCD Metric Carousel: Auto-rotates 8 screens every 3s: voltage, current, power, frequency, cost/hour, CO₂, safety margin, and load type
  • WiFi Manager UI: Full touchscreen provisioning with on-screen QWERTY keyboard, RSSI signal strength sorting, SPIFFS credential caching
  • Real-time Sensor Loop: PZEM-004T polled every 2 seconds; calculates 13 derived metrics including cost/hour, wasted power, safety margin, and CO₂ equivalent
  • Relay Protection Logic: 7 independent fault criteria (UV/OV/OC/OP/UF/OF/LPF) with hysteresis-based thresholds to prevent chattering; 5s auto-restore cooldown
  • Cloud Synchronization: Async HTTPS POST every 3s via WiFiClientSecure; includes AES-128-CBC cookie caching to bypass InfinityFree bot protection
  • OLED Status Display: Compact view of current time, relay state, and active trip reason — always readable in low-light conditions
  • Dual-Core Architecture: Core 1 (Main Loop @ 2s): sensor reads, protection eval, display updates | Core 0 (HTTP Task @ 3s): cloud upload and relay polling — synchronized via FreeRTOS semaphores

Metrics Engine (13 Real-Time Metrics)

The PZEM-004T v3.0 sensor is polled every 2 seconds, delivering raw readings (voltage, current, power, energy, frequency, power factor). These are processed to derive:

  • Apparent Power (S): V × I (VA)
  • Reactive Power (Q): √(S² − P²) (VAR)
  • Load Type: Classified by power factor (resistive, mixed, inductive, reactive)
  • Cost/Hour: (P / 1000) × ₱13.8161/kWh
  • Wasted Power: S − P (W)
  • Safety Margin: ((16 − I) / 16) × 100 (%)
  • CO₂ Equivalent: energy × 0.6032 kg/kWh
  • Frequency Deviation: |f − 60| Hz

Constants: 220V nominal @ 60Hz, 16A circuit rating, ₱13.8161/kWh electricity cost (Meralco), 0.6032 kg CO₂/kWh emission factor.

Relay Protection System

The relay is normally energized. It trips OFF when any fault occurs and auto-restores after a 5-second cooldown. All 7 conditions use hysteresis to prevent relay chatter:

Protection Trip Threshold Restore Threshold
Undervoltage < 195 V ≥ 205 V
Overvoltage > 250 V ≤ 240 V
Overcurrent > 15 A ≤ 13.5 A
Overpower > 3200 W ≤ 3000 W
Under-frequency < 57 Hz ≥ 58.5 Hz
Over-frequency > 63 Hz ≤ 61.5 Hz
Low Power Factor < 0.50 (P > 100W) ≥ 0.65 or P < 50W

Audio & Multimedia System

  • I2S Audio DAC Interface: GPIO 26 (BCLK), GPIO 27 (LRC), GPIO 5 (DOUT) for speaker output
  • Alert Sounds: Per-fault audio classification — distinct tones for overcurrent, undervoltage, overvoltage, etc.
  • TTS Integration: Google Gemini voice synthesis generates BMO-personality recommendations in Adventure Time style
  • SD Card Audio System: Async prefetch downloads cached TTS and alert WAV/MP3 files during idle periods (semaphore-protected access)
  • WiFi Camera Integration: Optional ESP32-CAM JPEG snapshots can be uploaded for occupancy-based relay automation

Dual-Core Real-Time Design

Core 1 (Main Loop — Priority: Sensor + Protection):

  • Polls PZEM-004T sensor every 2 seconds
  • Evaluates 7 protection criteria with hysteresis
  • Updates TFT animated face, OLED, and rotating LCD screens
  • Plays audio alerts immediately when faults are detected
  • Protected access to shared `HttpPayload` struct via FreeRTOS semaphore

Core 0 (HTTP Task — Priority: Cloud Sync):

  • POSTs timestamped readings to cloud every 3 seconds
  • Polls relay command queue from backend (async control queue)
  • Implements 3-minute timeout handling for lost connections
  • Includes AES-128-CBC cookie caching to bypass InfinityFree protections
  • Non-blocking: firmware continues operating locally if cloud is unreachable

Web Backend & Dashboard

Backend Stack: PHP 7.2+, MariaDB 11.4 (InfinityFree) | Frontend: Vanilla JavaScript, Chart.js 4.4.1, no frameworks

Four-Page Dashboard at pmo.infinityfree.me:

  • Dashboard: Relay control panel, AI analytics card (A–F rating + ₱ bill forecast + 3 recommendations), WiFi status, 1s refresh
  • Monitor: 13 live metric cards with dynamic color thresholds (green → amber → red based on safety margins)
  • History: 4 real-time Chart.js line charts (Voltage, Power Triangle, Current/PF dual-axis, Frequency) + 30-row scrollable sensor log
  • NILM Page: Appliance registry with 30-second training modal, live detection with confidence scores, signature management

AI Analytics: Google Gemini 1.5 Flash generates daily energy reports at 6 AM & 6 PM (Asia/Manila timezone). Aggregates daily stats (avg voltage, avg power, total energy, avg PF, cost projection) into letter grade (A–F), month-end ₱ bill estimate, and 3 household-specific recommendations. Results cached with 5.5-hour TTL to avoid redundant API calls.

NILM (Non-Intrusive Load Monitoring): Users register 30-second power signatures for any appliance. Detection algorithm runs exhaustive subset search against live metrics with confidence scoring. Returns matched device names and subset composition — no additional sensors needed.

Key Metrics & Impact

  • 13 Electrical Metrics: Voltage, current, power, energy, frequency, power factor, apparent power (VA), reactive power (VAR), load type, cost/hour, cost/month, wasted power, safety margin
  • 7 Protection Criteria: Independent trip/restore thresholds with hysteresis — UV (195/205V), OV (250/240V), OC (15/13.5A), OP (3200/3000W), UF (57/58.5Hz), OF (63/61.5Hz), LPF (0.50/0.65)
  • Real-Time Cloud Sync: 1 Hz web dashboard refresh with stale-connection detection (auto-lock after 5 missed updates)
  • AI Recommendations: Twice-daily Gemini-powered energy tips personalized to household usage patterns, delivered via text and BMO-voiced audio
  • Offline Resilience: Full protection and display functionality without internet; graceful cloud re-sync when connection restored
  • NILM Algorithm: O(2^n) exhaustive subset search with confidence scoring for practical performance on <10 appliances
